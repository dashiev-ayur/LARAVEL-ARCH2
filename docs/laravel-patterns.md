# Паттерны Laravel в этом проекте

Краткий обзор того, **как устроен бэкенд** в репозитории: не учебник по Laravel, а срез практик, которые уже используются здесь.

## Слой приложения: сценарии и порты

- **Use case (интерактор)** — классы в `app/Application/**/UseCases/` с методом `execute(...)`, инкапсулируют один бизнес-сценарий (создание, обновление, список и т.д.).
- **Порт (интерфейс)** — контракты в `app/Application/**/Ports/` (например, запись и чтение списков разделены на разные порты). Use case зависит от абстракции, а не от конкретной реализации.
- **Адаптеры** — сервисы в `app/Services/**`, реализующие порты и работающие с Eloquent, вспомогательными сервисами домена и т.п. Привязка интерфейса к классу выполняется в `AppServiceProvider::register()` через `$this->app->bind(...)`.

Итог: тонкий контроллер вызывает use case; инфраструктура подменяется через контейнер.

## HTTP-слой

- **Form Request** — валидация и авторизация запросов вынесены в `app/Http/Requests/**`; в сценарии передаётся только результат `$request->validated()`.
- **API Resource** — единообразная JSON-обёртка ответов API (`app/Http/Resources/`), в т.ч. коллекции через `::collection()`.
- **Контроллер** — маршрутизация к use case и формирование ответа (коды статусов, заголовок `Location` при создании ресурса и т.д.), без дублирования бизнес-логики.
- **Именованные маршруты** — для генерации URL (в т.ч. в заголовках ответа).

## Модели и доменные данные

- **Eloquent** как основная модель персистентности (`app/Models/`).
- **Enum** для ограниченных наборов значений (роли, статусы и подобное) с приведением через `casts()` на модели.
- **Observer** — реакция на жизненный цикл модели (например, при `updated` подготовка данных и диспатч доменного события).
- **Трейты в `app/Concerns/`** — переиспользуемое поведение (генерация уникальных slug, правила валидации и т.п.) без раздувания базовых классов.

## События, слушатели и очереди

- **Доменное событие** — класс в `app/Events/`, диспатч из observer или другого кода по смыслу операции.
- **Слушатели** — `app/Listeners/`; часть побочных эффектов может уходить в **Job** (`app/Jobs/`) для асинхронной обработки.
- **Явная регистрация слушателей** в `EventServiceProvider::$listen` при **отключённом автодискавери** событий — чтобы не дублировать подписки и контролировать порядок/набор обработчиков.

## Авторизация и доступ

- **Policies** — проверка прав на действия с моделями (например, команды).
- **Кастомные правила валидации** в `app/Rules/` для нетривиальных проверок, вынесенных из Form Request.
- **Middleware** — сквозная логика (контекст команды в URL, членство в команде, оформление Inertia и т.д.).

## Аутентификация (Fortify)

- **Действия Fortify** в `app/Actions/Fortify/` — создание пользователя, сброс пароля и т.д.
- **Кастомные HTTP-ответы** в `app/Http/Responses/` для согласованного поведения после логина, регистрации, 2FA и т.п.
- Настройка возможностей и маршрутов — через конфиг Fortify и провайдер приложения.

## Уведомления

- **Notification-классы** для писем и каналов доставки (приглашения в команду и аналогичные сценарии).

## Документация API (OpenAPI)

- Описания операций и схем вынесены в `app/Http/OpenApi/**` (атрибуты `OpenApi\Attributes`), синхронизируются с реальными контроллерами вручную; генерация артефакта — стандартной командой проекта.

## Конфигурация приложения (Laravel 11+ стиль)

- Точка входа **`bootstrap/app.php`**: маршруты, middleware по умолчанию, health endpoint.
- **`AppServiceProvider`** — биндинги портов, дефолты дат (`CarbonImmutable`), защита от разрушительных команд БД в production, настройка сложности пароля через `Password::defaults()`.

## Фронт (Inertia + React)

- **Inertia middleware** — общие props и корневой шаблон.
- **Маршруты `Route::inertia(...)`** для страниц без отдельного контроллера там, где достаточно статической страницы с данными из middleware.

## Тестирование

- **Pest** для feature- и unit-тестов; сценарии API, событий, генерации Swagger и т.д. лежат в `tests/`.

---

При добавлении нового домена разумно **копировать существующий срез** (`Application/<Domain>/UseCases`, `Ports`, `Services`, Form Requests, Resource, при необходимости Events/Listeners), чтобы не плодить второй архитектурный стиль.
